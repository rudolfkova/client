# Туман (`fog.kage`) и блюр (`blur.kage`) — инструкция для интеграции в клиент

Документ для агента: как устроены шейдеры, контракты с Go (Ebitengine), **порядок слоёв «блюр под туманом»** (сначала размытие картинки/сцены, затем туман поверх), и как накладывать туман **только на пустые тайлы** в игре, сохраняя единое поле шума.

Референс в репозитории: `example/fog/main.go` (цепочка **картинка → blur → fog** на экран).

---

## 1. Порядок слоёв: блюр под туманом

Имеется в виду **композиция по глубине**: нижний слой — уже размытая картинка (или сцена), **верхний** — полупрозрачный туман.

1. **Собрать слой без тумана** в `*ebiten.Image` того же размера, что и финальный кадр (например `scratch`: карта, спрайты, фон — как у вас принято).
2. **Блюр** в отдельный `*ebiten.Image` (`blurred`):
   - **Один проход на весь кадр** (проще, быстрее): `DrawRectShader(w, h, blurShader, opts)` с `opts.Images[0] = scratch`, где `scratch` ровно `w×h` (см. ограничение Ebiten ниже).
   - **По тайлам с непрерывным результатом** (как в `example/fog/main.go`): для каждого тайла в **отдельный буфер** копируется прямоугольник `scratch` с **ореолом** (padding ≥ радиуса сэмплов блюра), на этом буфере вызывается `DrawRectShader(cw, ch, …)` с `Images[0]` того же размера `cw×ch`, затем в `blurred` переносится только центральная область размера тайла. Иначе Ebiten не даст подставить полный `scratch` в вызов 32×32.
3. **Вывести размытый слой** на цель: `screen.DrawImage(blurred, nil)` (или в ваш offscreen).
4. **Туман** — поверх: `screen.DrawRectShader(w, h, fogShader, fogOpts)` с альфа-смешением (туман в `fog.kage` отдаёт **премультиплицированный** RGBA).

Туман **не** размывается блюром: блюр только на «чистой» сцене под ним. Если нужен размытый туман — это отдельный дизайн (другой порядок или второй проход).

## 3. Шейдер блюра (`blur.kage`)

### 3.1. Ограничение Ebiten (почему туман и блюр «по-разному» тайлятся)

У `DrawRectShader(width, height, …)` размер **каждого** непустого `options.Images[i]` **обязан** совпадать с `width×height` (иначе panic).

- **Туман** может рисковаться кусками 32×32 **без** текстуры-источника: шум считается из `dstPos` + uniform камеры.
- **Блюр** читает соседей через `imageSrc0At(srcPos ± offset)`. Если в `Images[0]` положить только обрезок 32×32 без соседей, на границах тайла не из чего сэмплить → **швы**. Чтобы блюр был **непрерывным** при тайловых вызовах, в `Images[0]` нужно подавать кусок **с ореолом** (halo), уже скопированный из полного `scratch` в Go (см. `drawBlurTiledPerTile` в `main.go`).

Шейдер `blur.kage` остаётся «локальным» по `srcPos`; непрерывность обеспечивает **данные** в буфере, а не `dstPos` внутри Kage (в отличие от тумана).

### 3.2. Uniform

```kage
var BlurPx float
```

Смещение сэмплов в **текселях** `Images[0]`. Чем больше `BlurPx`, тем шире ядро (в `main.go` — `blurStrengthPx`). Padding ореола в Go должен быть **не меньше** этого радиуса (в примере: `ceil(blurStrengthPx) + blurPadExtra`).

### 3.3. Полноэкранный вызов (один проход)

```go
bop := &ebiten.DrawRectShaderOptions{
    Uniforms: map[string]any{"BlurPx": float32(blurStrengthPx)},
    Images: [4]*ebiten.Image{},
}
bop.Images[0] = scratch // Bounds().Size() == image.Pt(w, h)
blurred.DrawRectShader(w, h, blurShader, bop)
```

Отдельный `blurred` (или иной dst), чтобы не писать поверх `scratch` в тот же проход, откуда читают сэмплы.

### 3.4. Тайловый вызов с ореолом (как в примере)

Для тайла `(tx, ty)` размером `(tw, th)`:

1. `pad` ≥ радиуса блюра; `ax := max(0, tx-pad)`, `bx := min(W, tx+tw+pad)` (аналогично `y`); `cw := bx-ax`, `ch := by-ay`.
2. В буфер `haloIn` размера `cw×ch` (через `SubImage` большого заранее выделенного буфера) скопировать фрагмент `scratch` со смещением `(-ax, -ay)`.
3. `DrawRectShader(cw, ch, blur, Images[0]=haloIn)` → `haloOut`.
4. Фрагмент `haloOut` с локальными координатами `(tx-ax, ty-ay)` размера `tw×th` нарисовать в `blurred` в точку `(tx, ty)` через `DrawImage` + `GeoM`.

Память: два буфера максимум `min(W, tileSize+2*pad) × min(H, tileSize+2*pad)` на пару in/out.

### 3.5. Настройка силы блюра

`blurStrengthPx` в Go и веса в `blur.kage`. Сильный блюр + мелкая сетка тайлов = много проходов за кадр; для продакшена часто оставляют **один** полноэкранный blur после сборки `scratch`.

## 4. Шейдер тумана (`fog.kage`) — uniform из Go

```kage
var Time float
var EdgeFade float
var CamX float
var CamY float
```

Передача из Go (одинаковые значения на все вызовы тумана в кадре, если тайлов несколько):

```go
Uniforms: map[string]any{
    "Time":     float32(…),     // секунды, монотонно
    "EdgeFade": float32(…),     // 0…1: у границы с «землёй» в клиенте; в демо без маски — 1
    "CamX":     float32(…),     // камера в px, как у тайлов
    "CamY":     float32(…),
},
```

Остальные параметры вида (плотность, дрейф, альфа) — **литералы в функциях** внутри `fog.kage` (`fogNoiseCells`, `fogDriftX`/`Y`, `fogAlphaScale`), не отдельные uniform-поля.

---

## 5. Почему туман «цельный» при рисовании по тайлам (игра)

В `fog.kage` шум считается в **мировых пикселях** (через камеру):

```kage
worldPx := dstPos.xy + vec2(CamX, CamY)
ds := imageDstSize()
p := worldPx * fogNoiseCells() / ds + vec2(Time * fogDriftX(), Time * fogDriftY())
```

- `dstPos` — пиксель в целевом `Image`.
- `CamX`/`CamY` должны совпадать со смещением камеры отрисовки карты на этот кадр.
- Так туман **не ломается** по сетке 32×32 при отдельных `DrawRectShader` на каждый пустой тайл с одним и тем же `dst` и одними и теми же uniform.

**Не** заменять эту схему на чисто локальные UV тайла (`srcPos / 32`), если нужен единый узор по миру.

## 6. Интеграция в игру: туман только на пустые тайлы

### 6.1. Порядок вместе с блюром

Типичный кадр:

1. Нарисовать **карту и объекты** в буфер `scratch` (или сразу на экран, если без offscreen).
2. **Опционально:** `scratch → blurred` — либо **одним** `DrawRectShader(W,H,…)` на весь кадр, либо **тайлами с ореолом** (раздел 3.4), затем `screen.DrawImage(blurred)`; без блюра: `screen.DrawImage(scratch)`.
3. Поверх экрана (уже с размытой или неразмытой сценой): для каждой **пустой** видимой клетки — `DrawRectShader(32, 32, fog, …)` с `GeoM.Translate(tx, ty)` и теми же `Time` / `EdgeFade` / `CamX` / `CamY`.

Туман всегда **после** того слоя, который вы хотите видеть под ним (включая размытый, если блюр включён).

### 6.2. Геометрия одного тайла тумана

```go
var gm ebiten.GeoM
gm.Translate(float64(tx), float64(ty))
op := &ebiten.DrawRectShaderOptions{
    GeoM: gm,
    Uniforms: map[string]any{
        "Time": t, "EdgeFade": fade, "CamX": camX, "CamY": camY,
    },
}
dst.DrawRectShader(tw, th, fogShader, &op)
```

`tx`, `ty` — в системе координат того же `dst`, что и карта (с учётом камеры, как у остальных тайлов).

### 6.3. Два `*ebiten.Shader`

При старте: `NewShader` для `fog.kage` и при использовании блюра — для `blur.kage`. В кадре только `DrawRectShader` / `DrawImage`.

---

## 7. Камера и размер `dst`

- Туман и блюр завязаны на размер **того изображения**, которое является `dst` при вызове (и на `Images[0]` для блюра).
- Offscreen вьюпорта другого размера — нормально, если вся цепочка (карта → blur → вывод → fog) согласована по размерам и камере.

---

## 8. Настройка вида тумана (только `fog.kage`)

| Функция | Смысл |
|---------|--------|
| `fogNoiseCells()` | Плотность шума (через `worldPx * … / imageDstSize()`): больше → мельче зерно. |
| `fogDriftX()`, `fogDriftY()` | Скорость дрейфа от `Time`. |
| `fogAlphaScale()` | Сила альфы (плотность тумана). |

Цвет: `fogColor` в `Fragment` — при необходимости вынести в такие же функции.

---

## 9. Частые ошибки

1. **Туман до блюра** — тогда размывается и фон, и туман; нужен порядок из раздела 1.
2. **Blur без отдельного `blurred`** — запись в тот же `Image`, из которого читают сэмплы, даёт артефакты.
3. **Размер `Images[0]` ≠ размер прямоугольника** в `DrawRectShader` для блюра — panic в Ebiten.
4. **Тайловый blur без ореола** — полный `scratch` нельзя передать в вызов 32×32; обрезок 32×32 без соседей даёт **швы** на границах тайлов.
5. Разный **`Time`** между вызовами тумана в одном кадре — шов анимации.
6. **`CamX`/`CamY` не совпадают** с камерой карты — туман «плывёт» относительно тайлов.
7. **Не премультиплицировать RGB тумана на α** — в `fog.kage` уже `vec4(fogColor*a, a)`; не откатывать к `vec4(fogColor, a)`.

## Краткий чеклист для агента

1. Собрать сцену в буфер (`scratch`), при необходимости **blur** в `blurred`: либо один полноэкранный `DrawRectShader` с `Images[0]=scratch`, либо тайлы **с ореолом** (раздел 3.4); затем вывести `blurred` на экран.
2. **Поверх** нарисовать туман (`DrawRectShader` полноэкранно или по пустым тайлам) с uniform `Time`, `EdgeFade`, `CamX`, `CamY`.
3. Один раз скомпилировать `fog.kage` и при необходимости `blur.kage` через `ebiten.NewShader`.
4. Визуал тумана крутить в `fogNoiseCells` / `fogDrift*` / `fogAlphaScale` в `fog.kage`; силу блюра — `BlurPx` из Go или веса в `blur.kage`.
