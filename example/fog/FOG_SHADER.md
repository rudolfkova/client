# Шейдер тумана (`fog.kage`) — инструкция для интеграции в клиент

Документ описывает, как устроен шейдер, какие есть контракты с Go (Ebitengine), и **как правильно накладывать туман только на пустые тайлы**, чтобы картинка оставалась одним непрерывным полем, а не «ломалась» по сетке.

Референс-реализация: `main.go` в этой же папке (`example/fog`).

---

## 1. Назначение

- Процедурный полупрозрачный туман (шум + вертикальный градиент по яркости).
- Предполагается отрисовка **кусками** (например 32×32 пикселя на тайл), но визуально туман должен выглядеть **как один слой на всём целевом изображении** (без повторения одного и того же паттерна в каждой ячейке).

---

## 2. Файлы

| Файл | Роль |
|------|------|
| `shaders/fog.kage` | Исходник Kage, компиляция через `ebiten.NewShader([]byte)`. |
| `main.go` | Изолированный пример: сетка 32 px, опциональная «дырявая» маска для теста. |

В игровом клиенте имеет смысл положить копию или общий embed того же `fog.kage` туда, где собирается рендер карты.

---

## 3. Uniform (обязательно из Go)

В шейдере объявлено:

```kage
var Time float
```

В Go при каждом `DrawRectShader` (или хотя бы раз на кадр для всех вызовов) нужно передавать:

```go
Uniforms: map[string]any{
    "Time":     float32(…), // секунды, монотонно растущее значение
    "EdgeFade": float32(…), // 0…1, в игровом клиенте — смягчение у границы с землёй (слой 0); в примере можно всегда 1
    "CamX":     float32(…), // смещение камеры в px (как у тайлов) — шум в мировых координатах: worldPx = dstPos + Cam
    "CamY":     float32(…),
},
```

Рекомендация: одно и то же `Time` на весь кадр для всех тайлов с туманом (например `float32(time.Since(start).Seconds())`), иначе анимация по тайлам разойдётся. **`CamX`/`CamY`** должны совпадать с камерой отрисовки карты на этот кадр.

Остальные «ручные» параметры (плотность, скорость, альфа) заданы **внутри `fog.kage` функциями с `return`** — **не** передаются uniform-ами (см. раздел 8).

---

## 4. Почему туман «цельный» при рисовании по тайлам

Критический фрагмент шейдера:

```kage
uv := (dstPos.xy - imageDstOrigin()) / imageDstSize()
```

- `dstPos` — координата **пикселя в целевом изображении** (`*ebiten.Image`), куда идёт отрисовка (часто экран или буфер камеры).
- `imageDstSize()` / `imageDstOrigin()` — размер и смещение **этого же** destination-изображения.

Раньше UV брались от **экранного** `dstPos` (туман «лип» к окну). В игровом клиенте шум считается в **мировых пикселях** `worldPx = dstPos.xy + vec2(CamX, CamY)` (те же `CamX`/`CamY`, что у тайлов), нормировка по `imageDstSize()` сохраняет плотность «зерна» при разном разрешении. При скролле камеры узор едет вместе с картой.

**Важно для агента:** для одного непрерывного поля по-прежнему один и тот же `dst` и одинаковые `CamX`/`CamY` на все вызовы `DrawRectShader` в кадре.

## 5. Интеграция: туман только на пустые тайлы

### 5.1. Порядок слоёв (типично)

1. Нарисовать карту (тайлы с контентом).
2. Для каждой **пустой** ячейки сетки, которая попадает в видимую область, нарисовать **один** `DrawRectShader` размером тайла (например 32×32) в **экранные** (или «камерные») координаты этой ячейки.
3. Не вызывать шейдер там, где тайл не пустой — туман там не появится.

### 5.2. Геометрия вызова (как в `main.go`)

Для тайла в пикселях верхнего левого угла `(tx, ty)` и размера `(tw, th)` (обычно 32, 32; на краю окна может быть меньше):

```go
var gm ebiten.GeoM
gm.Translate(float64(tx), float64(ty))
op := &ebiten.DrawRectShaderOptions{
    GeoM: gm,
    Uniforms: map[string]any{"Time": t},
}
dst.DrawRectShader(tw, th, shader, &op) // dst — тот же Image, что и для карты в этом проходе
```

- `tx`, `ty` должны быть **в системе координат того же `dst`**, для которого шейдер использует `imageDstSize()` (часто уже после учёта камеры: либо рисуете сразу на экран в экранные координаты, либо в offscreen того же размера что и логика UV).

### 5.3. Один `*ebiten.Shader` на кадр

Создание: один раз `ebiten.NewShader(embeddedFogKageBytes)` при старте; в кадре только `DrawRectShader` с общим указателем.

---

## 6. Камера и не-полноэкранный dst

- Шейдер «привязан» к **размеру и началу того Image, который является текущим dst** при вызове.
- Если рендер карты идёт в **отдельную текстуру** меньше экрана (например, суб-регион), то `imageDstSize()` будет от этой текстуры — туман будет непрерывным **внутри неё**. Это нормально, если все тайлы с туманом рисуются в тот же Image.
- В **игровом клиенте** шум считается в мировых пикселях через uniform **`CamX`/`CamY`** (см. раздел 3). В изолированном `example/fog` для теста передаются нули — туман ведёт себя как привязанный к экрану.

---

## 7. Настройка вида (правка `fog.kage`, литералы в функциях ниже)

В начале файла после объявлений uniform — функции, в которых меняются **только литералы в `return`**:

| Функция | Смысл |
|---------|--------|
| `fogNoiseCells()` | Плотность шума (через `worldPx * … / imageDstSize()`): **больше** → мельче «зерно». |
| `fogDriftX()`, `fogDriftY()` | Скорость дрейфа (множители к `Time` в формуле смещения). |
| `fogAlphaScale()` | Общая сила альфы: **больше** → плотнее туман (разумный диапазон обычно 0…1, можно чуть выше по вкусу). |

Цвет тумана сейчас зашит в `Fragment`: `fogColor := vec3(0.75, 0.75, 0.8)` — при необходимости вынести в такие же функции по тому же шаблону.

---

## 8. Тестовый пример vs продакшен

В `main.go` функция `fogTileOmittedForTest` **намеренно пропускает** часть плиток, чтобы визуально проверить непрерывность тумана. В клиенте **не использовать**: вместо этого условие «пустой тайл / не пустой» из логики карты.

---

## 9. Частые ошибки

1. **Разный `Time` на соседних вызовах в одном кадре** — шов по фазе анимации.
2. **Рисование тумана в другой `*ebiten.Image`**, чем тот, для которого задумана единая UV-сетка, без понимания размеров — шум не совпадёт с ожидаемым «экраном».
3. **Замена UV на локальные координаты тайла** — туман станет одинаковым на каждом тайле (артефакт «плитки»).
4. **Забыть залить фон** под туманом — полупрозрачный туман смешивается с тем, что уже в буфере (в примере сначала `screen.Fill`).
5. **Не премультиплицировать RGB на α** — возвращать `vec4(fogColor, a)` вместо `vec4(fogColor*a, a)` даёт «молочную» заливку по клетке при малых α (Ebiten ждёт premultiplied RGBA).

---

## 10. Запуск примера

Из корня модуля `client`:

```bash
go run ./example/fog
```

Убедиться, что `go:embed` путь к `shaders/fog.kage` корректен при переносе файлов.

---

## Краткий чеклист для агента

1. Встроить/прочитать байты `fog.kage`, один раз `ebiten.NewShader`.
2. На кадр вычислить `Time` (float32, секунды).
3. После основной отрисовки карты: для каждой **пустой** видимой клетки — `DrawRectShader` с `GeoM` в координатах **того же** `dst`, куда пойдёт карта.
4. Uniform: только `Time` (пока не расширяли контракт).
5. Визуал подкручивать в функциях `fogNoiseCells`, `fogDriftX/Y`, `fogAlphaScale` в `fog.kage`.
