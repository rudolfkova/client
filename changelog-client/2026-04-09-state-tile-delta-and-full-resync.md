# Клиент: `state` — дельта тайлов (`tile_updates`) и полный снимок по таймеру

## Суть

Сообщение **`type: "state"`** по-прежнему приходит **каждый игровой тик**. Поля **`players`** и **`tick_at`** заполняются **всегда**.

**Тайлы** больше не дублируются целиком на каждом тике:

- На тиках **полного сброса** сервер шлёт **`tiles`** — полный массив (как раньше по смыслу), плюс **без** `tile_updates` за этот тик.
- На остальных тиках поле **`tiles` отсутствует** в JSON (если нет полного снимка); при изменениях тайлов за тик приходит **`tile_updates`** — массив операций.
- Если за тик **не было** ни полного снимка, **ни** изменений тайлов — **нет** ни `tiles`, ни `tile_updates` (клиент не трогает кэш тайлов).

**Игроки** не дельтились: весь список **`players`** каждый тик, как и раньше.

## Интервал полного снимка тайлов

- **TOML** `game-service`: `tile_full_sync_interval` — строка длительности (как `tick_rate`), например **`"1s"`**.
- **Окружение**: `TILE_FULL_SYNC_INTERVAL` — то же, что понимает **`time.ParseDuration`** в Go (например `500ms`, `2s`). Ненулевое положительное значение **перекрывает** TOML.
- **Первый тик жизни процесса** game-service (а не «первое сообщение каждого WebSocket») даёт полный снимок тайлов: у движка сброшен таймер полной синхронизации. Клиент, подключившийся позже, может сначала получить `state` без `tiles` — до ближайшего полного тика или пока не придут `tile_updates`.

## Схема `tile_updates`

Элемент массива:

```json
{ "op": "upsert", "tile": { "x", "y", "layer", "rotation", "texture", "blocks", "instance_args?" } }
```

или

```json
{ "op": "remove", "x": 0, "y": 0, "layer": 0 }
```

Константы **`op`** в **`pkg/gamekit/wire.go`**: `StateTileUpdateUpsert` = **`"upsert"`**, `StateTileUpdateRemove` = **`"remove"`**.

- **`upsert`**: логически «в этой клетке на этом слое сейчас такой тайл» (сервер перед спавном чистит слой; клиенту достаточно применить один upsert на `(x,y,layer)`).
- **`remove`**: очистка слоя в `(x,y,layer)` без замены. Координаты **`0`** сериализуются явно (без `omitempty` на полях удаления).

## Типы Go (`pkg/gamekit`)

- **`StatePayload.Tiles`** — указатель на слайс **`[]Tile`** с **`json:"tiles,omitempty"`**: при полном снимке для **пустого мира** приходит **`"tiles":[]`** (не `null`).
- **`StatePayload.TileUpdates`** — **`[]TileUpdate`**, `omitempty` если пусто.

## Рекомендация клиенту

1. Держать локальную карту тайлов по ключу **`(x, y, layer)`** (или свою структуру слоёв).
2. На каждом `state`: применить **`players`/`tick_at`**; если есть **`tiles`** — **заменить** локальный набор тайлов снимком; иначе применить **`tile_updates`** по порядку.
3. Раз в **`tile_full_sync_interval`** (и при первом кадре) сверяться с полным **`tiles`** — защита от пропущенных сообщений.

## Регрессия по репозиторию

Ломающее изменение только в **форме JSON `state`**: клиенты, которые всегда читали **`tiles`**, должны перейти на логику выше.
