# Патч: `click_layer` = 0 из Go, пустой `instance_args`, лимит 64 KiB

Краткая дельта к **`2026-04-13-tile-instance-args-interact-click.md`**.

---

## Изменения

### 1. `InteractIntent.click_layer` (wire / `pkg/gamekit`)

- JSON-тег **`click_layer`**: снят **`omitempty`**, чтобы **`json.Marshal` из Go** сериализовал **`"click_layer": 0`** при указателе на `0` (явный нулевой слой).
- Если слой не нужен: в JSON **не включайте ключ** или передайте **`null`** при разборе на сервере это **`nil`** → логика «максимальный слой среди подходящих тайлов».
- **Цена:** при `Marshal` структуры с `ClickLayer: nil` в JSON попадёт **`"click_layer":null`** (если вы сериализуете этот тип в JSON). Для WebSocket клиент обычно шлёт только нужные поля — ключ опускают.

### 2. `instance_args` (тайлы, `spawn_tile`, `world_spawn_tile`)

- Пустой объект **`{}`** после нормализации **не хранится** и **не попадает** в **`state.tiles[]`** (эквивалент отсутствия поля, меньше трафика).
- Максимальный размер сырого JSON объекта: **`gamekit.MaxTileInstanceArgsJSONBytes`** = **65536** байт. Больше — поле **отбрасывается** (как при неверном типе), без паники.

---

## Краткий ответ клиентскому агенту

1. **`click_layer`:** для явного слоя **0** в JSON всегда можно отправить **`"click_layer": 0`**. В Go при использовании **`encoding/json.Marshal`** на **`gamekit.InteractIntent`** нулевой слой теперь уходит в JSON; если слой не задан — **не включайте ключ** (или `null`), сервер выберет тайл с **максимальным** `layer` среди тайлов с `texture == item_def_id`.

2. **`instance_args` в `state`:** в массиве **`tiles`** поле есть **только** если объект **непустой**; **`{}`** и слишком большие / неверные значения на входе **не** появляются в состоянии.

3. **Лимит:** объект **`instance_args`** на wire не длиннее **64 KiB** (`gamekit.MaxTileInstanceArgsJSONBytes`); иначе сервер игнорирует поле при нормализации.

Константа и нормализация: **`pkg/gamekit/tile_instance_args.go`**, типы: **`pkg/gamekit/wire.go`**.
