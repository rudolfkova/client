# Клиент: `instance_args` на тайлах и клик по клетке для `interact`

Дельта к **`changelog-client/2026-04-13-content-interact-ws.md`**. Старые клиенты без новых полей продолжают работать (поля опциональны в JSON).

---

## 1. `spawn_tile` → `state.tiles[]`

В **`payload`** (`gamekit.TileSpawnIntent`) добавлено опциональное поле:

| JSON-ключ       | Тип   | Смысл |
|-----------------|-------|--------|
| `instance_args` | object | Произвольный JSON-**объект** на экземпляр тайла. `null` / отсутствие — нет данных. |

Если пришло **не** объект (массив, строка, число) — сервер **не падаёт**: поле **отбрасывается**, тайл создаётся без `instance_args`.

В исходящем **`state`** у элемента **`tiles[]`** (`gamekit.Tile`) то же поле **`instance_args`** (object), только если на сервере оно непустое. Пустой объект `{}` допустим и уходит как `{}`.

Персистентность: поле хранится в ECS на компоненте **`TileTexture`** вместе с `name` (текстура); попадает в **ark-serde** снимок мира / world-service как часть того же компонента. Старые снимки без ключа при загрузке дают пустое поле.

---

## 2. `interact` — клик по клетке

В **`gamekit.InteractIntent`** добавлены **опциональные** поля (все с `omitempty` в Go, на wire можно не слать):

| JSON-ключ     | Тип | Смысл |
|---------------|-----|--------|
| `click_x`     | int | Клетка клика. |
| `click_y`     | int | Клетка клика. |
| `click_layer` | int | Явный слой тайла. |

Правила:

- Режим клика включается **только если заданы оба** `click_x` и `click_y`. Если передан только один — клик **не** используется, сценарий идёт как раньше **только по `item_def_id`** (без чтения тайла).
- **Резолв тайла:** среди тайлов в `(click_x, click_y)` выбираются те, у которых **`texture` (имя текстуры) == `item_def_id`**, и у записи **`catalog.items[item_def_id].interact`** в каталоге есть сценарий.
- **`click_layer`:** если передан — берётся ровно этот слой; иначе — тайл с **максимальным** `layer` среди подходящих.
- Если клик включён, но подходящего тайла нет — **`interact` полностью игнорируется** (без `reject`).

**Договорённость с контентом:** id предмета в каталоге и имя текстуры на карте должны **совпадать строкой** (клиент спавнит рычаг с `texture: "lever_01"` и шлёт `item_def_id: "lever_01"`).

---

## 3. Мерж аргументов в сценарий

Перед вызовом `Runner` для каждого шага по-прежнему: `merge(база_для_тика, args_шага)` — **args шага сценария перекрывают всё**.

База для тика собирается так (**последний перекрывает ключи предыдущего**):

1. **`interact.args`** из каталога для этого `item_def_id`;
2. поверх — **`instance_args`** с выбранного тайла (если режим клика и тайл найден; иначе только каталог);
3. дальше как раньше — **args шага** из JSON сценария.

Итого: **`каталог < тайл < шаг сценария`**.

При **без** клика: в базу входит только каталог (как раньше по смыслу).

## 4. Каталог для редактора (не рантайм)

В **`interact`** записи предмета опционально:

| JSON-ключ                         | Смысл |
|-----------------------------------|--------|
| `editor_instance_args_example`    | Пример объекта для UI редактора. **Сервер сценариев не читает.** |

## 5. `world_spawn_tile` в сценарии

В `args` шага можно передать **`instance_args`** (object) — попадёт в `TileSpawnIntent` при спавне с тайла (нормализация та же, что у WS `spawn_tile`).

---

## Критерий проверки (из ТЗ)

1. Редактор шлёт **`spawn_tile`** с **`instance_args`** → в следующем **`state`** тот же объект на соответствующем элементе **`tiles[]`**.
2. Клиент шлёт **`interact`** с **`item_def_id` + `click_x` + `click_y`** (+ при необходимости **`click_layer`**) → в сценарии видны смерженные args (после настройки сценария и тайла).
3. Клиент **без** новых полей — без изменений поведения.
